# News Workflow Quick Start Guide

Get started with the news post workflow system in 5 minutes.

## Setup

The news workflow system is already integrated into your application. No additional setup required!

## Quick Overview

**Workflow**: Author writes → Designer adds images → Boss reviews → Publish

## Getting Started

### 1. Create Your First Post (Author)

1. Navigate to **News** in the sidebar
2. Click **"Create Post"** button
3. Fill in the form:
   - Title (required)
   - Content (required)
   - Excerpt (optional but recommended)
   - Category
   - Tags
   - SEO settings
4. Click **"Save Draft"**
5. Your post is now in **Draft** status

### 2. Submit for Design

1. Find your draft post in the list
2. Click the **eye icon** to open workflow modal
3. Click **"Submit for Design"**
4. Post moves to **Awaiting Design** status
5. Designers are notified

### 3. Add Images (Designer)

1. View posts with **"Awaiting Design"** status
2. Click the **eye icon** on a post
3. Click **"Start Design Work"**
4. Edit the post to add images:
   - Click **"Upload Images"**
   - Select multiple images
   - Preview and arrange
5. Return to workflow modal
6. Click **"Submit for Review"**

### 4. Review & Approve (Boss)

1. View posts with **"Pending Review"** status
2. Click the **eye icon** to review
3. Check content and images
4. Choose action:
   
   **To Approve:**
   - Add optional comment
   - Click **"Approve Post"**
   
   **To Request Changes:**
   - Write detailed feedback (required)
   - Click **"Request Revision"**
   - Author and designer will see your feedback

### 5. Publish

1. Find approved posts
2. Click the **eye icon**
3. Choose:
   - **"Publish Now"** - Goes live immediately
   - **"Schedule Post"** - Select date/time for future publication

## User Roles

### Author
- Creates posts
- Edits drafts
- Submits for design
- Makes revisions

### Designer  
- Adds images
- Designs layout
- Submits for review

### Boss/Reviewer
- Reviews posts
- Approves or requests revisions
- Publishes posts

### Admin/Super Admin
- Full access to everything
- Can override workflow

## Useful Features

### View Modes
- **All Posts**: See everything
- **My Posts**: Only your posts
- **Pending My Action**: Posts waiting for you

### Filters
- Filter by status
- Search by title/content
- Sort by date, title, or status

### Comments
- Add comments at any stage
- Communicate with team
- Track feedback history

### Revision History
- See all changes
- Track who did what
- Version numbers

## Common Workflows

### Standard Flow
```
Draft → Awaiting Design → In Design → Pending Review → Approved → Published
```

### Revision Flow
```
Pending Review → Needs Revision → (Author/Designer fixes) → Pending Review → Approved
```

### Quick Publish (Admin)
```
Draft → (Admin edits and adds images) → Published
```

## Tips

1. **Authors**: Write clear, complete content before submitting for design
2. **Designers**: Add high-quality images that match the content
3. **Boss**: Provide specific, actionable feedback when requesting revisions
4. **Everyone**: Use comments to communicate and ask questions

## Keyboard Shortcuts

- Click post title to view details
- Eye icon = Open workflow
- Edit icon = Edit post
- Trash icon = Delete post

- Check **History** tab to see what happened

## Example Scenario

**Monday**: Author writes article about new technology
- Status: Draft

**Tuesday**: Author submits for design
- Status: Awaiting Design

**Wednesday**: Designer adds images and infographics
- Status: In Design → Pending Review

**Thursday**: Boss reviews and requests minor changes
- Status: Needs Revision
- Feedback: "Please add more context in paragraph 3"

**Friday**: Author updates content, designer resubmits
- Status: Pending Review → Approved → Published
